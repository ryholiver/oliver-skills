# 内容类型指南

本文档说明拆解文档中每种内容类型的写法、适用场景和格式要求。

---

## 内容类型总览

| 类型 | 适用层级 | 主要作用 |
|------|---------|---------|
| 信息卡片 | 产品概况 | 快速传递结构化基础信息 |
| 竞争格局表 | 市场层 | 对比多个竞品的关键维度 |
| 商业模式拆解 | 商业层 | 呈现变现逻辑和护城河 |
| 正向流程图 | 用户层 | 描述用户完整操作路径 |
| 逆向流程图 | 用户层 | 描述异常/回退路径 |
| 评测打分表 | 用户层 | 量化评测结果 |
| 架构类型判断 | 技术层 | 定性判断系统架构 |
| 全局上下文Schema | 技术层 | 推导共享数据结构 |
| Agent架构图 | 技术层 | 展示Agent组成和关系 |
| Agent卡片 | 技术层 | 详细描述单个Agent |
| 提示词骨架 | 技术层（可选）| 推导关键Agent的Prompt结构 |
| 模型路由表 | 模型层 | 分析模型使用策略 |
| 数据资产清单 | 基础层 | 梳理数据资产和飞轮 |
| 面试口径卡 | 附录 | 面试准备的快速参考 |
| 复刻技术路线图 | 附录 | 复刻时的实现规划 |

## 正向流程图

**作用**：描述用户完成核心任务的完整步骤序列。

**格式要求**：
- 使用ASCII流程图（兼容性好，不依赖渲染器）
- 每个节点标注主体：`[用户]` 还是 `[系统]`
- 标注 `← [用户决策点N]` 在需要用户选择的地方
- 标注等待时间（如知道的话）

**示例**：
```
[用户] 上传商品图（支持JPG/PNG，<20MB）
    ↓
[系统] 图像分析（主体检测 + 背景分割）约3秒
    ↓
[系统] 展示分割预览，3个预设场景推荐
    ↓ ← [用户决策点1] 选择场景 / 输入自定义描述 / 上传参考图
[系统] 场景生成（并行生成4个候选）约15-25秒
    ↓
[系统] 展示4个候选结果 + 对比查看工具
    ↓ ← [用户决策点2] 选择满意结果 / 重新生成 / 进入精修
[用户] 下载（单张/批量）/ 继续操作
```

**关键注意**：
- 不要遗漏异步等待的步骤
- 每个"系统"步骤背后可能是一个Agent，要有意识地标注

## 逆向流程图

**作用**：描述出错、不满意或中途放弃时的处理路径。

**格式**：与正向流程图类似，但强调分叉点和回退路径

**示例**：
```
[用户] 对生成结果不满意
    ├── 选项1：重新生成（相同参数）
    │       ↓
    │   [系统] 使用相同Prompt重新采样 → 4个新候选
    │
    ├── 选项2：调整参数重新生成
    │       ↓ ← [用户决策点] 修改哪些参数？
    │   [系统] 更新上下文 → 重新规划 → 重新生成
    │
    └── 选项3：进入精修模式
            ↓
        [用户] 使用局部修复/替换工具 → 手动调整
```

## Agent 卡片

**作用**：标准化描述单个Agent的完整画像。

**格式模板**：
```markdown
### Agent：[名称]（推导版）

**职责定义**：[一句话说明这个Agent负责什么]

**触发时机**：[在什么条件下被调用]

**输入**：
- `[字段名]`：[描述] `[体验推导/合理推测]`
- ...

**处理逻辑**：
1. [第一步做什么]
2. [第二步做什么]
3. ...

**工具集**：
| 工具 | 用途 | 推断依据 |
|------|------|---------|
| 图像分割API | 提取前景主体 | 输出精度和速度符合专用分割模型特征 |
| ... | | |

**输出**：
- `[字段名]`：[描述]
- ...

**范式判断**：Plan-and-Execute / ReAct / 纯Workflow
**判断依据**：[为什么这么判断]

**规则与边界**：
- 必须：[约束1]
- 禁止：[约束2]
- 如果X则Y：[条件逻辑]

**置信度**：高 / 中 / 低
**主要不确定点**：[哪里最不确定]
```

## 全局上下文 Schema

**作用**：推导贯穿所有Agent的共享数据结构。

**推导方法**：
1. 列出每个Agent的所有输入字段
2. 列出每个Agent的所有输出字段
3. 找出被多个Agent引用的字段 → 候选全局字段
4. 按类别整理为Schema

**格式**：YAML代码块，带详细注释

```yaml
# 全局上下文 Schema
# 版本：推导版 v1.0 | 置信度：中
# 推导依据：基于用户层流程观察 + 合理推断

session_context:
  # === 元数据 ===
  session_id: "string"          # 唯一会话标识
  created_at: "ISO8601"
  user_tier: "free|pro|team"    # 影响质量上限和速度

  # === 输入资产 ===
  input:
    raw_image_url: "string"     # 原始上传图 [ImageAnalysisAgent 读]
    product_type: "string"      # 商品品类 [ImageAnalysisAgent 写]
    extracted_features:         # 提取的产品特征 [ImageAnalysisAgent 写]
      - feature: "string"
        confidence: "float"

  # === 用户偏好（跨步骤累积）===
  preferences:
    scene_style: "string"       # [ScenePlannerAgent 写，后续Agent读]
    custom_prompt: "string"     # 用户自定义描述
    reference_image: "string"   # 参考图（可选）

  # === 生成结果 ===
  generation:
    job_id: "string"
    status: "pending|running|done|failed"
    candidates: []              # 4个候选结果
    selected_index: "int"       # 用户选择

  # === 质检记录 ===
  quality:
    passed_candidates: []       # 通过质检的候选
    rejection_reasons: []       # 被过滤的原因
```

## 竞品对比矩阵

**作用**：对比多个产品在关键维度上的差异。

**推荐维度**（根据赛道调整）：
- 核心功能定位
- 目标用户
- 技术深度（自研模型 vs 套壳）
- 定价策略
- 护城河类型
- 最强场景
- 最弱场景
- 数据飞轮

**格式**：
| 维度 | 产品A | 产品B | 产品C |
|------|-------|-------|-------|
| 定位 | ... | ... | ... |
| 技术深度 | 自研底模 ★★★ | 微调+工程 ★★ | 纯API封装 ★ |
| ... | | | |

## 评测打分表

**作用**：量化体验评测，为技术层判断提供依据。

**格式**：
| 维度 | 权重 | 得分(1-10) | 扣分原因 | 推断影响 |
|------|------|-----------|---------|---------|
| 生成质量 | 30% | 8 | 光影融合偶有问题 | Prompt调优空间 |
| 生成速度 | 20% | 7 | 批量时有排队 | 并发架构瓶颈 |
| 交互流畅度 | 20% | 9 | - | - |
| 异常处理 | 15% | 6 | 失败无明确原因 | 错误处理不完善 |
| 可控性 | 15% | 7 | 精修工具有限 | MVP阶段合理 |
| **综合** | | **7.4** | | |

## 面试口径卡

**作用**：把拆解结论转化为面试现场可以直接使用的回答框架。

**格式**：
```markdown
### Q：请描述一下这个产品的技术架构

**回答框架**（STAR结构）：
"[产品名]是一个[一句话定位]。从技术架构角度看，它采用了[架构类型]——我判断这一点的依据是[2条观察]。核心是[主要Agent及职责]，之间通过[协作方式]串联。最有意思的设计决策是[1个亮点]，因为[理由]。当然，也存在[1个问题]，如果我来做PM会[改法]。"

**扩展话题**：
- 为什么用Multi-Agent而不是单Agent？
- 全局上下文如何设计的？
- 它的护城河在哪里？

**关键数据点**（记住这几个数字）：
- 生成时长：约X秒
- 市场规模：$Xbn
- 竞品分布：X家主要竞品
```

## 复刻技术路线图

**作用**：为想复刻该产品的团队提供实施规划参考。

**格式**：
```markdown
## 复刻技术路线图

### MVP边界（最精简可验证版本）
- ✅ 包含：[必须有的核心功能]
- ❌ 排除：[暂不实现的功能]

### 技术选型建议
| 模块 | 推荐方案 | 替代方案 | 难度 |
|------|---------|---------|------|
| 图像分割 | SAM2（Meta开源） | 商业API | 中 |
| 场景生成 | SDXL + ControlNet | Flux | 高 |
| ... | | | |

### 难点排序
1. [最难的]：[为什么难 + 建议解法]
2. [次难的]：...

### 成本估算（按月活1000用户）
- 模型API：~$X/月
- 存储：~$X/月
- 计算：~$X/月
- 合计：~$X/月

### 关键里程碑
- Week 1-2：[目标]
- Week 3-4：[目标]
- Month 2：[目标]
```
