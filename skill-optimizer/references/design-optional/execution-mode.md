# 可选模块：执行模式设计

**适用条件**：skill 涉及不可逆操作、文件修改、或需要用户在关键节点参与决策时读取本文件。

---

## 核心问题：Claude 的行为模型是什么？

设计执行模式前，直接调用 AskUserQuestion（不输出前置文字）：
- A：计划确认 —— Claude 先输出计划，用户确认后再执行
- B：自主执行 —— Claude 直接执行，完成后汇报结果

判断参考（有助于向用户说明推荐选项）：
- 有不可逆操作、步骤多、结果影响用户数据 → 推荐 A
- 步骤少且可逆、用户已明确授权、效率优先 → 推荐 B

---

## 如果选 A（计划确认）：计划用什么格式？

计划格式没有标准答案，取决于任务类型。直接调用 AskUserQuestion（不输出前置文字）：
- A：步骤列表（每步 + 预期产出）
- B：文件树变更预览（适合批量文件操作）
- C：根据任务自行设计一份格式

用户选 C，或无偏好 → Claude 根据任务特点设计格式，写入 SKILL.md 时附上示例。

**格式设计的判断依据**：
- 文件操作多 → 用文件树或 diff 格式（before/after 清晰）
- 步骤线性 → 用编号步骤列表
- 决策节点多 → 用带分支的流程格式
- 结果是一个文档 → 用"大纲预览"格式

---

## 用户覆盖（两种模式都应支持）

无论默认选哪种，都要在 SKILL.md 里留出口：
- 用户说"直接做" → 跳过确认，自主执行
- 用户说"先告诉我" → 切换到计划确认，无论默认是什么

---

## 在 SKILL.md 里的写法

```markdown
## 执行模式
默认：[计划确认 / 自主执行]（一句话说明原因）
[如果计划确认：计划格式为 X，示例如下：...]
用户可覆盖：说"直接做"跳过确认；说"先告诉我"输出计划
```

控制在 5-8 行，格式示例如有必要可放 references/ 文件里。
