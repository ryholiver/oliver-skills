# 可选模块：跨 Skill 调用设计

**适用条件**：skill 的工作流中有步骤需要调用或依赖另一个 skill 的能力时读取本文件。

---

## 四维分类

每个跨 skill 调用在设计时先明确这四个维度，再决定如何写：

**1. 关系类型**（决定数据如何交接）
- **参数型**：子 skill 输出是下一步的精确输入变量 → 写明"等待返回值 X，用于步骤 N 的 Y 参数"
- **上下文型**：子 skill 输出丰富后续判断，非精确参数 → 写明"结合返回结果决定步骤 N 的方向"
- **插曲型**：不影响父流程，用户临时偏离 → 父流程状态不变，插曲结束后提示继续
- **探索型**：可能影响也可能不影响，执行后由用户/Claude 决定是否修改原计划

**2. 阻塞性**（决定父流程是否暂停）
- 阻塞型：父流程必须等子 skill 完成 → 在步骤说明里写"此处暂停等待"
- 非阻塞型：父流程继续，子 skill 结果后续合并

**3. 上下文传递**（决定子 skill 需要什么信息）
- 需要完整上下文 → 调用前输出 context capsule（关键信息摘要）
- 只需特定参数 → 只传参数，不传背景
- 独立运行 → 直接触发，不需交接

**4. 触发来源**（决定是否要在 skill 里预先设计）
- 计划内调用 → 必须在工作流里写明
- 用户自发的插曲 → 不需要预先设计，只需在 SKILL.md 说明打断处理规则

---

## 写法示例

在 references/ 工作流文件的对应步骤里：

```markdown
## 步骤4：扫描已有文件（参数型·阻塞·需部分上下文）

需要文件名列表用于生成双向链接。
- 调用 vault-scanner skill，传入：vault_path={已确认路径}
- 等待返回：文件名列表
- 降级方案（vault-scanner 不可用）：执行 `find {vault_path} -name "*.md" | head -50`
- 获得列表后继续步骤5
```

---

## 降级方案原则

每个计划内的跨 skill 调用都应设计降级方案：
- 子 skill 不存在 → 用内置工具/脚本替代
- 子 skill 失败 → 告知用户，提供手动操作路径
- 子 skill 返回空结果 → 定义空结果时的默认行为

插曲型调用不需要设计降级（因为不影响父流程）。
