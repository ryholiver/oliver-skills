# 多插件合集规范

管理多个相关的 plugins（如 claude-code-plugins-plus）。

---

## 目录结构

```
my-plugins/
├── plugins/
│   ├── category-a/
│   │   └── plugin-a/
│   │       ├── .claude-plugin/plugin.json
│   │       ├── skills/
│   │       │   └── my-skill/
│   │       │       └── SKILL.md
│   │       └── README.md
│   └── category-b/
│       └── plugin-b/
│           └── ...
├── marketplace/
│   └── marketplace.json
└── README.md
```

---

## plugin.json

每个 plugin 都有自己的 plugin.json：

```json
{
  "name": "plugin-a",
  "description": "插件 A 描述",
  "version": "1.0.0",
  "author": { "name": "你的名字" },
  "license": "MIT"
}
```

---

## marketplace.json

```json
{
  "name": "my-plugins",
  "description": "我的插件集合",
  "plugins": [
    {
      "name": "plugin-a",
      "description": "插件 A 描述",
      "version": "1.0.0",
      "author": { "name": "你的名字" },
      "license": "MIT",
      "category": "category-a"
    },
    {
      "name": "plugin-b",
      "description": "插件 B 描述",
      "version": "1.0.0",
      "author": { "name": "你的名字" },
      "license": "MIT",
      "category": "category-b"
    }
  ]
}
```

---

## 分类

- `ai-ml` - AI 与机器学习
- `api-development` - API 开发
- `business-tools` - 商业工具
- `community` - 社区插件
- `database` - 数据库工具
- `devops` - DevOps 与基础设施
- `performance` - 性能工具
- `productivity` - 生产力工具
- `security` - 安全工具
- `testing` - 测试工具

---

## README.md

根目录的 README 说明整个合集：

```markdown
# My Plugins

我的插件集合。

## 插件列表

- [plugin-a](plugins/category-a/plugin-a/) - 插件 A 描述
- [plugin-b](plugins/category-b/plugin-b/) - 插件 B 描述
```
