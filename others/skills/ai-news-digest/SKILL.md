# AI 资讯聚合 Skill

## 概述

本 Skill 帮助用户：
1. 从多源渠道采集 AI 最新资讯（英文 + 中文）
2. 按话题/重要性整理分类
3. 输出**可直接发布**的 HTML 可视化资讯卡片

---

## 第一步：明确本次任务类型

在开始前，先确认用户意图属于以下哪种模式：

| 模式 | 触发词示例 | 说明 |
|------|-----------|------|
| **A. 每日/每周摘要** | "出一期周报"、"今天 AI 圈" | 全面覆盖，按热度排序 |
| **B. 话题深度追踪** | "追踪 Agent 进展"、"大模型最新动态" | 聚焦某个垂直方向 |
| **C. 竞品/行业监控** | "OpenAI/Anthropic 有什么新动作" | 聚焦特定公司或产品线 |
| **D. 发布内容生成** | "帮我写公众号"、"做 Newsletter" | 输出即发布的完整内容 |

如果用户没说清楚，默认走 **A. 每日摘要** 模式。

---

## 第二步：信息采集

### 2.1 信息源库

读取用户的信息源配置（见 `references/sources.md`），按优先级搜索。

**默认信息源（无自定义时使用）：**

#### 英文源
- **Twitter/X 关键账号**：@OpenAI, @AnthropicAI, @GoogleDeepMind, @Meta AI, @MistralAI, @scaling01, @karpathy, @ylecun
- **Reddit**：r/MachineLearning, r/LocalLLaMA, r/artificial
- **arXiv**：cs.AI, cs.LG, cs.CL（最新论文）
- **官方博客**：openai.com/blog, anthropic.com/news, deepmind.google/research, ai.meta.com/blog

#### 中文源
- **36kr**：36kr.com/information/AI
- **知乎**：AI 话题热榜
- **虎嗅/极客公园**：AI 频道

### 2.2 搜索策略

根据模式调整搜索词：

```
模式A（摘要）：  "AI news [today/this week]" + "AI 资讯 [今日/本周]"
模式B（话题）：  "[topic] latest developments" + "[话题] 最新进展"  
模式C（竞品）：  "[company] announcement" + "[公司] 发布"
模式D（发布）：  同A/B/C，但后续生成步骤会调整为发布风格
```

**执行：** 使用 `web_search` 工具执行 3-6 次搜索，覆盖中英文。重要新闻用 `web_fetch` 获取完整内容。

## 第三步：内容整理与分类

将采集到的资讯按以下分类整理：

### 分类体系

```
🔥 重磅发布    — 新模型/产品/重大功能上线
🧠 研究进展    — 论文/技术突破
🏢 公司动态    — 融资/并购/人事/战略
🛠️ 工具&开源   — 开源项目/开发者工具
📊 行业观察    — 报告/数据/趋势分析
💡 产品观察    — AI 产品体验/案例
```

---

### 重要性评分规则（显式矩阵）

每条资讯从以下四个维度打分，加总后映射到星级。

#### 维度 A：话题相关度（0-3分）
读取 `references/sources.md` 中的「我的关注优先级」，对照打分：

| 情况 | 分值 |
|------|------|
| 命中用户标记为「核心」的话题 | 3 |
| 命中用户标记为「关注」的话题 | 2 |
| 命中用户标记为「泛读」的话题 | 1 |
| 与用户偏好无关 | 0 |

> 若用户未配置偏好，默认：大模型发布=3，Agent/工具=2，行业政策=1。

#### 维度 B：观点/信息增量（0-3分）

这是最核心的维度，**时间新旧不是判断标准，信息增量才是**。

| 情况 | 分值 |
|------|------|
| 提出全新视角，或首次披露数据/事实 | 3 |
| 对已知事件有实质性深化（新数据、新分析、反驳观点） | 2 |
| 同一话题已有报道，本篇仅重复已知内容 | 1 |
| 纯转载/聚合，无额外信息 | 0 |

**关键规则：同话题存在多篇时，按增量排序，增量高的优先展示，不论发布时间先后。** 一篇三天前的深度分析，若观点比今天的新闻稿更有价值，排名更高。

#### 维度 C：影响范围（0-2分）

| 情况 | 分值 |
|------|------|
| 影响整个 AI 行业格局或多个领域 | 2 |
| 影响某个细分领域或特定用户群 | 1 |
| 仅影响单一产品/公司内部 | 0 |

#### 维度 D：信源权威度（0-2分）

| 情况 | 分值 |
|------|------|
| 官方一手发布（公司博客/发布会/论文） | 2 |
| 权威媒体独家报道（CNBC、The Verge、机器之心等） | 1 |
| 二手转载、聚合站、匿名消息 | 0 |

---

### 总分 → 星级映射

| 总分（满分10） | 星级 | 含义 |
|--------------|------|------|
| 8-10 | ⭐⭐⭐ | 重磅，必读 |
| 5-7  | ⭐⭐  | 值得关注 |
| 2-4  | ⭐   | 参考了解 |
| 0-1  | 不收录 | 信息价值太低，直接丢弃 |

---

每条资讯提取：
- **标题**（中文，20字内）
- **一句话摘要**（50字内，说清"是什么+为什么重要"）
- **重要性评分**（按上方矩阵计算）
- **原文链接**
- **来源标签**（英文/中文/官方）

---

### 黑马探测：识别新兴话题

**在对所有资讯完成评分后，额外执行一次扫描：**

专门检查「维度A得0分」的资讯（即不在用户已知关注列表里的内容），判断是否命中以下信号：

| 信号 | 说明 |
|------|------|
| **多源共振** | 2个以上独立信源在48小时内报道同一事件或词汇 |
| **权威首发** | 来自官方博客或一线媒体，且话题在用户关注列表里从未出现过 |
| **概念新词** | 搜索结果中反复出现一个用户列表里没有的新词/新概念 |
| **社区爆发** | Reddit热帖/知乎急速上榜，讨论量异常高 |

**判断规则：命中任意2条以上信号 → 标记为 🆕 新兴话题**

新兴话题不参与常规评分排序，单独收集，在HTML卡片中作为独立分区展示（见第四步）。

每期报告结束时询问用户：
> "本期发现了 X 个新兴话题，是否把「[话题名]」加入你的关注列表？"

---

## 第四步：生成 HTML 可视化卡片

**这是核心输出步骤。** 读取 `references/html-template.md` 获取卡片模板规范，生成完整 HTML。

### 卡片规格

- 宽度：375px（移动端优先）或 750px（宽屏版）
- 风格：深色科技感 或 简洁白色（根据用户偏好）
- 必须包含：
  - 期号 + 日期标题栏
  - 分类色块标签
  - 每条资讯卡片（标题 + 摘要 + 来源 + 链接）
  - 页脚（"由 AI 资讯 Skill 生成"）

### 输出文件

将生成的 HTML 保存为 `/mnt/user-data/outputs/ai-digest-[YYYYMMDD].html`

---

## 第五步：确认 & 迭代

生成后询问用户：
1. 是否需要调整某条资讯的摘要？
2. 是否有遗漏的重要新闻（用户可补充链接）？
3. 是否需要切换卡片风格（深色/浅色/宽屏）？

---

## 特殊功能：信息源管理

当用户说"管理我的信息源"、"添加信息源"、"我想追踪 XX"时：
→ 读取并编辑 `references/sources.md`，更新用户的自定义信息源库。

---

## 注意事项

- **版权**：摘要必须是改写后的内容，不直接复制原文，每条资讯附原文链接
- **时效性**：优先展示 48 小时内的新闻，超过 7 天的内容标注"旧闻"
- **去重**：同一事件的多个报道合并为一条，选最权威来源
- **语言**：输出统一为中文（英文资讯翻译摘要，保留英文原标题）
