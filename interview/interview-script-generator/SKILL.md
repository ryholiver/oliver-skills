# 面试话术生成器

## 核心定位

输入一个面试问题或场景 → skill 自主解构问题、调取素材、设计讲述方案、基于偏好档案生成话术。
不满意时，用维度池定位问题，调整后重新生成。
每次满意的结果自动沉淀到样例库和偏好档案。
话术修正过程中的误解自动记录到误解库。

---

## 文件结构

```
interview-script-generator/
├── SKILL.md
└── references/
    ├── _index.md              # 样例索引，每次触发先读
    ├── _preference.md         # 偏好档案，每次触发必读
    ├── samples/               # 样例文件，按需读取
    └── misreads/
        ├── _misread-index.md  # 误解索引，每次修正前先读
        └── MR001-xxx.md       # 误解记录文件
```

**加载顺序**：
1. 每次触发必读 `_preference.md`
2. 读 `_index.md` 判断是否有相关样例 → 按需读具体样例文件
3. 生成话术前读 `_misread-index.md`，检查是否有已知的同类误解

## 执行流程

### Step 1：解构问题

拿到面试问题后，先自主分析：
- 这个问题表面在问什么
- 面试官真正想考察什么能力/特质
- 这个问题的陷阱或常见错误答法是什么
- 适合用什么话术功能回应（建立印象/传递信息/引发追问/关闭话题）

### Step 2：调取素材

读取 `_index.md`，判断是否有相关样例可以参考。
如果用户没有提供素材，主动询问：「这个问题你有什么经历或判断可以用？」
如果用户提供了素材，自己判断哪些可用、怎么用。

### Step 3：设计讲述方案

在生成话术前，先输出结构设计：

```
【讲述方案】
问题本质：面试官在考察 ___
结构走法：___ → ___ → ___
能力暴露点：在第___层透出，方式是___
钩子设计：埋在___位置，内容是___
基调：___
```

方案确认后再生成话术，避免方向跑偏后大改。

### Step 4：生成话术

基于偏好档案中的默认值生成，不需要用户逐一填写维度。

### Step 5：诊断与迭代

用户不满意时，用维度池辅助定位：

> 「是结构走法的问题，还是语感/口吻的问题，还是钩子没埋对，还是能力点没透出来？」

定位到具体维度后调整重新生成。

**误解检测**：如果用户指出"这不是我要的"或"你理解错了"，立即：
1. 读 `_misread-index.md` 检查是否有同类误解
2. 有则补充到对应文件；没有则新建误解记录
3. 基于正确理解重新生成

### Step 6：沉淀

用户确认满意后：
1. 将话术存入 `references/samples/` 新建文件
2. 更新 `_index.md` 添加索引条目
3. 如果本次调整暴露了新的偏好信号，更新 `_preference.md`

## 偏好档案更新原则

**「明确不要」只能来自两个来源，不能推测：**
1. 用户明确说了「不要X」
2. 用户多次修正同一个方向，从修正行为里提炼

**「惯用句式」和「惯用结构走法」**可以从满意样例里提炼，因为这是正向信号，从结果反推是可靠的。

**「风格默认值」**从多个样例的共同特征里提炼，单个样例不足以定义默认值。

## 维度池（诊断坐标）

不是每次都填，不满意时用来定位问题所在。

### 内容层
- **内容边界**：只讲什么，不讲什么
- **铺垫量**：直接结论 / 一句背景+结论 / 完整结构
- **颗粒度**：停在战略层 / 落到执行层 / 穿插两层
- **时间锚点**：重心在过去 / 现在 / 未来
- **信息层次**：表层结论 / 透出底层逻辑 / 完整推理链
- **情境还原度**：抽象概括 / 带场景感有画面
- **讲述结构**：STAR / 问题-判断-行动 / 结论-依据-例子 / 现状-洞察-方案 / 不套框架

### 意图层
- **能力意图**：想让面试官看到哪种能力
- **话术功能**：建立印象 / 传递信息 / 引发追问 / 关闭话题
- **话术隐含诉求**：争取信任 / 争取认同 / 争取下一轮
- **与JD对齐显性度**：不提 / 隐约呼应 / 直接点出
- **对面试官预期的管理**：满足预期 / 超出预期 / 重新定义问题

### 场景与关系层
- **触发场景**：主动展示 / 被动回答 / 追问回答
- **与对方的关系预设**：懂行同行 / 需被说服的决策者 / 评估者
- **话术水位**：刚好回答 / 留七分说三分 / 超额给信息
- **收尾与钩子设计**：自然收住 / 埋钩子（位置+内容）/ 反问传球

### 风格与语感层
- **口吻/人称距离**：对话 / 陈述 / 汇报
- **情绪底色**：兴奋 / 笃定 / 平静 / 谦逊
- **确定性程度**：板上钉钉 / "我的判断是" / 留有余地
- **专业术语密度**：全程行话 / 混合口语 / 白话讲清
- **类比/故事密度**：纯逻辑 / 用比喻 / 用小故事
- **身份感**：执行者 / 思考者 / 决策者 / 合作者
- **固定表达句式**：惯用的开场/过渡/收尾句式

### 自我呈现层
- **自我位置**：我是主角 / 我在解题 / 我在共同分析
- **主动性信号**：我在驱动 / 我在响应 / 我在协作
- **我与团队的边界**：说"我" / 说"我们" / 功劳归属处理
- **立场稳定性**：全程一致 / 呈现认知弧线（曾经→转变）
- **反脆弱信号**：要不要呈现踩坑/犯错/遇阻
- **压力感知**：从容 / 有紧迫感 / 在化解压力
- **能量方向**：向外输出展示 / 向内收敛回应

### 边界处理层
- **知识边界的处理**：回避 / 承认不知道 / 说出不确定性
- **比较基准**：对标行业 / 对标同龄人 / 对标上一份工作 / 不做比较
- **长度与密度**：点到为止 / 完整表达

## 首次使用：初始化流程

如果 `_preference.md` 为空或不存在，进入初始化：

1. 请用户提供背景素材（简历/经历/项目简述）
2. 请用户提供1-2段觉得满意的历史话术（如有）
3. 从样本中提炼初始偏好档案（只提炼惯用句式和结构走法，不推测不要什么）
4. 写入 `_preference.md`
5. 告知用户：「偏好档案已建立，后续使用会持续迭代」

如果用户没有样本，直接生成3个风格差异明显的版本，用排除法校准初始偏好。
