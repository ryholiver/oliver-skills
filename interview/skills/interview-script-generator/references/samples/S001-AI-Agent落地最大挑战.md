# 问题：AI Agent 落地最大的挑战是什么？

**场景**：综合型 AI PM 面试 / 也可用于主动展示趋势敏锐度
**话术功能**：传递信息 + 引发追问
**能力意图**：系统性思维、PM 衡量视角、趋势敏锐度
**讲述结构**：反直觉判断 → 根本原因 → 解法 → 衡量方式 → 趋势

---

## 话术

Agent 落地最大的挑战，我觉得不是模型够不够聪明，而是任务漂移——Agent 在执行长任务的过程中，会逐渐偏离最初的目标，做着做着就跑偏了。

要解决这个问题，靠 Agent 自己监督自己是不可靠的，因为没有可靠的基准。更有效的方式是从外部注入两样东西：一是标准知识库作为判断基准，二是明确的规则约束它的行为边界——不靠自律，而是让它物理上越不了界。

但光解决问题还不够，作为 PM 还要能衡量它。我认为最重要的指标不是报错率，而是采纳率——工程师有多少比例真正用了 Agent 给的建议。一个不出错但没人用的 Agent，对业务没有价值。

最后一点，这个方向的市场我觉得会持续扩大——因为 AI Coding 工具让代码产出越来越快，部署频率越来越高，反而需要更多的监控和治理。Harness 这类产品不会被替代，需求只会更大。

---

## 话术演变记录

**起点**：从零开始，用户对 AI Agent 了解有限，通过问答逐步推导出四个观点

**关键变化**：
- 衡量指标从「报错率」调整为「采纳率」→ 报错减少不等于有人用，采纳率才是业务价值的真实体现
- 结尾加入趋势判断 → 从「解决问题」升级为「判断方向」，展示敏锐度

**最终版本对了，因为**：四个观点全是用户自己推导出来的，有逻辑链条，能被追问 defend，不是背书
