# 问题：挑一个AI结合度比较高的拆解一下

**场景**：主动展示 / 项目深度拆解
**话术功能**：建立印象 + 传递信息
**能力意图**：用户洞察、结构性判断、技术方案选型、数据驱动验证
**讲述结构**：用户洞察 → 问题本质判断 → 方案比较 → 选择依据 → 结果 → 更大的视野

---

## 话术

我做的第一个功能是设计方向探索。

我访谈了5个设计师，新人和老手都有。她们都在用AI，但都没有真正用起来，得到的内容太泛，落不到业务决策上。

我当时的判断是：这是结构性问题。通用大模型没有设计领域的知识，给不出有密度的判断，只能给普适性建议。换个人用、换个方式提问都解决不了。

我考虑了两个方向——RAG结合设计领域数据，或者图生图做视觉探索。

图生图的优势是具象，能直接看到效果，但问题是：方向没收敛之前图出来也没有参照；它只能展示长什么样，解释不了为什么这个方向对；操作上要反复调参、精细控制，改一个方向就要重来一遍。

而RAG+大模型的优势是：方向探索需要的是推理——为什么这个方向可行、有什么依据、怎么取舍。文字可以承载逻辑链，图像不能。结合领域数据之后，输出的是有依据的判断，而不是视觉猜测。所以我选了RAG+大模型。

新人设计师能在2小时内锁定设计方向，方案评审通过率从0提升到40%。

另外，图像探索这个方向Lovart这类工具已经做得很完善了，自己做不划算。我认同其创始人说的一句话：要做产品的上游而不是下游。文字方向探索恰恰是图像生成工具的上游——方向探索完已经有足够的信息生成图像提示词了，图生图的流程是自然会打通的。
