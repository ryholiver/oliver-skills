# 上下文管理与兜底机制

> 本文件仅在上下文过长、出现记忆模糊或跨会话恢复时读取。正常流程无需加载。

---

## 兜底机制一览

| 风险场景 | 触发条件 | 兜底操作 |
|---|---|---|
| 深聊中原文细节模糊 | 需要引用原文验证某个说法 | 从 cleaned JSON 中读取对应时间戳片段，不做全量加载 |
| 跨会话原文丢失 | 新会话开始，无原文上下文 | 读取 `<文档名>_cleaned.json` + `<文档名>_知识地图.md` |
| 跨会话进度记忆清零 | 新会话检测到已有知识地图文件 | 展示进度恢复看板（格式见 knowledge_map_format.md）|
| 大文档知识提取不准 | 文档超过 60 分钟录音 / 清洗后 >200 段 | 分批读取：先提取话题列表，再逐组深入 |
| 对话过长，早期内容被压缩 | 感觉知识点描述与之前不一致 | 重新读 `<文档名>_知识地图.md`，以文件为准，不以记忆为准 |

---

## 核心原则

**原始文档和 cleaned 文件住在工作区文件系统，不住在对话上下文里。**

- 需要时按需读取（用时间戳或片段索引定位），不做全量加载
- 知识地图 .md 文件是进度的唯一可信来源，记忆不如文件可靠
- HTML 文件是进度的可视化镜像，由代码同步更新，不由记忆维护

---

## 时间戳索引读取示例

当需要验证某个知识点的原文依据时，不读整个 cleaned JSON，只读对应时间段：

```python
import json

with open("文档名_cleaned.json") as f:
    segments = json.load(f)["segments"]

# 只读 00:15:00 到 00:20:00 之间的片段
target = [s for s in segments
          if s["status"] == "keep"
          and "00:15" <= s["timestamp"] <= "00:20"]
```

将这几段内容粘贴到对话中，而非整个文件。
