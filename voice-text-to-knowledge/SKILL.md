# voice-text-to-knowledge

从带时间戳的多人语音转写稿中提取内行实践知识，通过深度对话转化为自己能输出的理解。

---

## 整体流程

```
文件来源判断
  → Step 1：预处理（代码）→ cleaned JSON
  → Step 2：知识提取（LLM）→ 知识地图 .md + HTML 知识地图（代码）
  → Step 3：用户选话题 → 深聊（由我主导节奏）
  → Step 4：知识结晶 → 存入 Obsidian + 代码更新 HTML 进度
  → Step 5：所有话题 ✅ → 提示清理临时文件
```

> **HTML 知识地图**在浏览器独立窗口中持续可见，深聊期间用户可随时切换查看全局进度。进度更新由代码完成，零 token。

---

## Step 1：预处理管道

**1a. 判断文件来源**

- 用户上传了文件 → 直接使用上传路径，进入 1b
- 没有上传 → 运行 `python scripts/scan_transcripts.py`，扫描授权目录，列出候选转写稿让用户选择（只有一个则直接确认）

**1b. 确定存储路径**

```bash
python scripts/setup_storage.py <文档文件名>
```

- 自动检测 iCloud Drive（Mac / Windows / Linux），找不到则回退到主目录
- 创建「转写稿处理」文件夹
- 如果已有处理过的文件 → 跳到进度恢复（Step 2 末尾），跳过 1c

**1c. 预处理文档**

```bash
python scripts/preprocess_transcript.py \
  --input <文档路径> \
  --output <存储路径>/<文档名>_cleaned.json
```

输出包含 `status: keep/skip` 标记的结构化 JSON。

---

## Step 2：知识提取 → 输出知识地图

读取 `_cleaned.json`，一次性完成 **ASR纠错 + 话题分组 + 知识标注**。

**格式规范和标注规则** → 读取 `references/knowledge_map_format.md`

知识地图保存为：`<工作区>/<文档名>_知识地图.md`

**Step 2 末尾必须执行：生成 HTML 知识地图**
→ 读取 `references/html_generation.md`，按指引生成 HTML 并在浏览器打开。

输出后**直接呈现给用户，不问"你想了解什么类型"**。

**进度恢复**（检测到已有知识地图文件时）
→ 读取 `references/knowledge_map_format.md` 末尾的「进度恢复看板格式」，展示后问从哪里继续。

---

## Step 3：深聊模式

用户选择知识点后进入深聊。**开始时运行进度更新脚本标为 🔄 进行中**（详见 `references/html_generation.md`）。

**我主导以下四步节奏，不跳步：**

1. **解释**：把知识点说清楚，补充背景，指出原对话中可能不准确的地方，加入自己的判断
2. **验证**：说明可信度边界——哪些经得起推敲，哪些需要打问号，有明显漏洞直接指出
3. **应用**：问用户「如果你要在自己的场景里用这个知识，你会怎么说/怎么用？」，等待回答
4. **追问**：根据回答做一件事：理解正确 → 追问更深问题；有偏差 → 指出并重新解释；说懂了 → 进入结晶

**可以进入下一个话题的信号**（满足任意一条）：
- 用户用自己的话复述出来了
- 用户说「懂了」「可以了」「换下一个」
- 用户问了超出该知识点范围的问题

**需要继续的信号**：复述有明显偏差 / 举不出应用例子 / 问「这是什么意思」

---

## Step 4：知识结晶

深聊结束后询问：「要把这个知识点记下来吗？」

如果是，整理为**用户视角的第一人称笔记**（不是对话记录，不是原文摘录，是「我理解的 XXX 是这样的」），包含用户提到的应用场景。

调用 `obsidian-knowledge-saver` skill 保存。

同步更新进度：
- `.md` 知识地图文件中该知识点改为 `✅ 已吸收 · [日期]`
- 运行 `update_progress.py` 更新 HTML → 详见 `references/html_generation.md`

---

## Step 5：文件清理

**触发条件**：知识地图所有知识点均为 `✅ 已吸收`

提示用户确认后，依次删除：

```bash
rm "<文档名>_cleaned.json"
rm "<文档名>_知识地图.md"
rm "<文档名>_知识地图.html"
rm "<原始文档>"
```

完成后提示：「文件已清理，知识已全部保存在 Obsidian 里。」

**不自动删除**，必须用户确认。用户可选择保留部分文件。

---

## 未定义判断处理

遇到现有标准无法覆盖的情况：

1. 暂停，标记：`🔍 未定义判断：[描述情况，说明哪条标准不适用]`
2. 按用户指示处理
3. 问「这个判断方式以后都这样吗？」
4. 用户确认普遍性 → 追加到 `references/judgment_standards.md`（标准8、9……）
