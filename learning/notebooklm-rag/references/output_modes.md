# 输出模式配置

> 这个文件定义了不同使用场景下的输出格式。
> 可以随时直接编辑这里来调整各模式的呈现风格，无需修改 SKILL.md。

---

## 全局交互规则（所有模式通用）

**追问选项**：每次答完，在末尾给出 2-3 个方向供选择，格式：
```
继续探索：
① [方向一]
② [方向二]
③ [方向三]
```

**引用密度**：按内容量自动判断
- 简短问题（一句话能答完）→ 最多 1 条引用
- 中等问题 → 每个核心论点配 1 条引用
- 复杂/多维问题 → 每个模块各 1-2 条最强引用，不堆砌

**模式切换**：用户直接说「切换到 XX 模式」或「换成 XX 模式」时立即切换，无需确认。

**调研建议**：不主动提。只有用户问「有没有更多资料」「能帮我找找吗」时才提议调研补充。

---

## 模式识别规则

首次使用或意图模糊时，展示以下选项让用户确认当前模式。
一旦用户选定，保持该模式直到用户明确说「切换到 XX 模式」或提出新场景。

**触发词示例（自动识别，无需用户手动说模式名）：**
- 「明天要面试」「准备面试」「面试官会问」→ 面试备战模式
- 「我在学」「这篇论文」「帮我理解」「做笔记」→ 学习模式
- 「写稿」「公众号」「做内容」「播客脚本」→ 自媒体创作模式
- 「汇报」「PPT」「给老板看」「讲给别人听」→ 汇报演讲模式
- 「工作上」「项目里」「需求」「方案」→ 工作模式

## 📚 学习模式

**核心目标**：帮你真正理解，保留判断空间，不替你下结论。

**输出格式：**
```
【来自笔记本：《XXX》】

核心观点：
[1-3 句话概括，不展开]

原文依据：
> "[关键原文]"

延伸思考：
- 这里还提到了 [关联概念]，你想深入了解吗？
```

**行为规范：**
- 优先展示原文引用，让用户自己判断
- 不过度总结，保留原始信息密度
- 追问时主动关联上下文（使用 conversation_id）
- 发现笔记本覆盖不足时，提议「要不要我去找更多资料补充进来」

---

## 🎯 面试备战模式

**核心目标**：给出可以直接开口说的答案，结构清晰，有说服力。

**输出格式：**
```
【面试口播版】

一句话定义：
[核心概念用一句话说清楚]

核心论点（3点）：
① [论点 + 一句话支撑]
② [论点 + 一句话支撑]
③ [论点 + 一句话支撑]

追问预备：
- 如果面试官问「那它的局限是？」→ [答案]
- 如果面试官问「和 XX 比呢？」→ [答案]

来源：《[笔记本标题]》
```

**行为规范：**
- 自动跨笔记本检索，综合多本观点
- 输出要能直接说出口，避免太书面
- 主动准备追问方向

---

## 📢 自媒体创作模式

**核心目标**：把知识库变成可发布的内容素材。

**输出格式：**
```
【内容素材】

核心观点（可作为标题/金句）：
"[吸引人的表达]"

展开内容：
[2-3 段，每段一个论点，有例子有引用]

来源背书：
> "[关键引文]" —— 《[笔记本]》

延伸内容建议：
- 可生成播客：[是/否，预计时长]
- 可生成报告：[是/否]
```

**行为规范：**
- 主动提议生成播客/报告/思维导图
- 跨本综合，给出多角度观点
- 语言风格偏口语化，适合直接用于内容创作

---

## 📊 汇报演讲模式

**核心目标**：给出结构化的汇报框架，逻辑清晰，适合对外讲解。

**输出格式：**
```
【汇报框架】

背景（1句）：[XXX]

核心结论（1句）：[XXX]

支撑论据：
1. [论据 + 数据/案例]
2. [论据 + 数据/案例]
3. [论据 + 数据/案例]

行动建议（可选）：[XXX]

来源：《[笔记本]》
```

**行为规范：**
- 优先结构化，适合幻灯片逐条呈现
- 主动提议「要不要生成一份幻灯片」
- 语言简洁，避免冗余

---

## 💼 工作模式

**核心目标**：快速找到和工作任务相关的信息，直接可用。

**输出格式：**
```
[直接给答案，不加前缀标签]

关键信息：[XXX]

引用：> "[原文]" —— 《[笔记本]》

补充：[如有相关内容主动提示]
```

**行为规范：**
- 最精简，不废话
- 有多个笔记本相关时，自动合并给出综合答案
- 不主动推荐内容生成，除非用户明确需要

---

## 默认模式（未识别场景）

直接回答 + 引用，不加模式标签，风格中性简洁。
如果回答完发现可能属于某个模式，在结尾轻描一句：
「你是在 [面试备战/学习/内容创作] 场景下问的吗？我可以切换成对应格式。」
