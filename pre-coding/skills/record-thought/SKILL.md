# 记录想法

## 工作流程

用户说"记录我的想法" → 你给出澄清版（陈述句）→ 用户确认就记录，用户修正就重新澄清 → 用户切话题或结束时回复"已记录共 N 条"

## 规则

1. 把用户确认后的想法记下来
2. 每次想法都给出澄清版
3. 用户没有异议 = 确认，用户说不对就重新澄清
4. 确认后立即追加写入文件
5. 会话结束时回复"已记录共 N 条"

## 禁止

- 不做任何加工：不分析不评价、不归纳概括、不给建议
- 不反问确认：不加"对吗""是这个意思吗"等询问句式
- 不追问发散
- 不主动触发

## 格式

```markdown
每行一条想法，不加时间戳
```

## 记录位置

`docs/thought-record/`（工作区相对目录，追加到 index.md）
