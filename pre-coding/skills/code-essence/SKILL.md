# Code Essence — 从混乱到有序

## 目的

有些代码写得完整、工程化、抽象层多 —— 但对于想理解它的人来说，这些"好东西"反而成了噪音。

这个技能的目标是：**给 AI 一段现有代码，AI 还你一个按理解层级剥离过的可读版本**。不是从零写最小实现，而是**从复杂中提取本质**。

## 这个技能的工作方式

这个技能定义的是**一个由你主导的递进理解过程**。AI 不做决策，只在你每次给出方向后，把目标代码剥离成你能看懂的骨架。

整个过程是：

```
① 你先给一个目标（一个文件 / 一个目录）
    ↓
② AI 调用 deep-dive-loop 去研究这个目标
   → 输出骨架 + 数据流 + 内部子功能列表
    ↓
③ 你看 AI 输出的骨架，理解它在做什么
    ↓
④ 你做决策：
   ├─ "我想继续看子功能 X" → 回到 ②，AI 用 deep-dive-loop 研究 X
   ├─ "关联的文件 Y 也看看" → 回到 ②，AI 研究 Y
   ├─ "我想全部看完" → AI 按顺序逐个研究每个子功能
   └─ "懂了，结束" → 过程结束
```

**AI 只负责"每次研究一个目标"，不负责决定研究什么、是否继续。** 研究什么、是否继续，全由你决定。

## deep-dive-loop 是什么

deep-dive-loop 是这个技能内部的工作模式，定义在 `workflow/deep-dive-loop.md`。它只做一件事：

> 输入一段代码 → 理解 → 剥离 → 输出骨架

每次你给出一个新方向，AI 就调用一次 deep-dive-loop。调用多少次、什么时候停，由你控制。

## 理解层级（输出格式）

AI 每次调用 deep-dive-loop 后，按 L1-L4 分层输出。L1 是所有人的起点，按需往外壳方向深入。

```
L1 ── 骨架 ← 所有人都从这里开始
L2 ── 骨架 + 外壳标注
L3 ── 骨架 + 外壳拆解
L4 ── 完整架构 + 迁移路径
```

### L1 — 骨架

所有人从这里开始。只保留最核心的执行路径，多层抽象内联成直线，变量/函数名重命名为大白话。没有日志、配置、校验、类型定义。附带中文数据流说明。

### L2 — 骨架 + 外壳标注

你已经看过 L1，想知道"原来这些位置有什么"。在 L1 基础上用注释标注外壳位置。

### L3 — 骨架 + 外壳拆解

你想了解每种外壳具体解决什么问题。在 L1 基础上把剥离的外壳分类列出。

### L4 — 完整架构 + 迁移路径

你想改造或重写。输出架构总览 + 每层外壳实现 + 模仿重写建议。

## 默认行为

第一次进入时，默认以 L1 输出目标代码的骨架 + 内部子功能列表。你看了之后告诉我下一步想看什么。

## 注意事项

- AI 不决定研究什么、是否继续——这些都由你做主
- deep-dive-loop 一次只研究一个目标，不递归，不擅自扩大范围
- 不认识的 API 或模式，优先查 Context7 或文档
- 代码很乱时，优先理清数据流，再理控制流
