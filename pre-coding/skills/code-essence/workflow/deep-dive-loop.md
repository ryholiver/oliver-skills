# Deep Dive Loop — 一次研究的完整过程

## 这个模式做什么

给你一段代码（或一个功能块），你把它剥离成我能看懂的骨架，输出给我。

## 输入

一段代码范围。可能是：
- 一个文件
- 一个功能块（指定行号范围）
- 一个目录
- 一个函数

## 准备工作

开始剥离前，先读取 `rule/剥离规则.md`，理解"什么该砍、什么该留"的判断标准。

## 工作步骤

### 第 1 步：理解输入

全量阅读输入范围内的代码，标记：
- 核心路径是什么（删掉就不跑的部分）
- 工程外壳是什么（能删掉的部分）
- 输入、输出、处理逻辑分别在哪

### 第 2 步：按剥离规则剥离

参照 `rule/剥离规则.md` 中的判断方法和强制删减清单，逐行检查代码。
核心原则：如果删掉某一行代码，核心功能仍然能跑，那这一行就不该留在骨架里。

### 第 3 步：自校验

自己检查有没有问题：
- 数据流完整吗（有输入→处理→输出的完整路径吗）
- 还有外壳残留吗（日志、校验、配置等没有砍干净的吗）
- 所有行在干同一件事吗（还是混了别的功能进来）
- 读起来逻辑通顺吗（从上到下能串起来吗）
- 能再简化吗（还能再砍掉什么吗）

不通过则最多调整 3 轮。第 3 轮仍不行就输出卡点说明。

### 第 4 步：输出

按技能要求的层级（L1-L4）输出剥离后的代码 + 数据流说明。

## 输出

一次研究的产出是：
1. 一句话总结：这段代码到底在做什么
2. 剥离后的代码骨架
3. 数据流说明
4. 内部子功能块的列表（供你选择下一步方向）
