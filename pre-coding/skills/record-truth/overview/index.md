# 关于本技能

从已完成的开发工作中萃取"上层事实"——代码本身没有明说，但代码中蕴含的约定、规则、技术规格、架构决策、配置约定，以及已经落地生效的技术约束。

它的产出不是变更记录，不是功能列表，不是未来规划。它只是记录：**现在项目里有什么是事实。**

---

## 架构概览

此 skill 包含三个环节，通过一条反馈回路连接：

```
环节一：素材采集与分析
  分析改动 → 提取候选事实

环节二：确认与产出
  格式化 → 确认 → 写入文件

环节三：修正（对产出不满意时进入）
  反复修改直到满意
  → 可选：从修正中提炼规则优化流程

```

核心原则：**每次修正不是浪费，而是训练信号**。环节三的每一次纠正都在积累数据，让下次更准。
