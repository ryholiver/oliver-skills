# Content Declutter

## 目标

去除转文稿中没有信息价值的表达（杂质），保留所有非杂质的原始内容，供后续处理使用。

**核心原则：**
- 不定义"什么是实质内容"——避免主观判断导致遗漏
- 不预设"保留什么"——由后续处理自行判断
- 不总结、不重写、不修改任何非杂质的表达
- 保留原始结构（时间戳、段落顺序等）

## 杂质类型

| 杂质类型 | 判定标准 |
|---------|---------|
| 开场白 | 引入性表达，无业务信息（如"在这个视频中我将向你们展示..."） |
| 确认话术 | 确认观众已知，不产生新信息（如"好的，我相信你们已经意识到..."） |
| 呼吁行动 | 说服/承诺性内容（如"我会保证你会..."） |
| 过渡语 | 仅承上启下，无实质信息（如"这就是如何将它付诸实践"） |
| 口头禅 | 无含义的语气词/口语（如"basically"、"literally"、"makes sense"、"almost"） |
| 情绪表达 | 个人感慨，非业务判断（如"我的头撞了墙"） |
| 重复确认 | 重复表达同一内容 |

杂质定义是动态的，可根据后续处理反馈补充新类型。

## 执行步骤

### Step 1: 接收原始转文稿

- 检测输入格式（带时间戳 / 不带时间戳）
- 按段落分割（空行分隔）
- 段落内按行保留结构

### Step 2: 扫描识别杂质

对每个段落/句子逐项检查是否匹配杂质类型。

### Step 3: 删除杂质

- 整行/整段是杂质的，直接删除
- 段落中混杂杂质和实质内容的，只删除杂质部分，保留实质内容
- 保留段落的原始顺序

### Step 4: 输出清洁文本

- 输出清洗后的文本
- 保留原始段落分隔
- 保留时间戳标记（如果原文有）

## 验收标准

- 所有已知杂质类型被删除
- 不产生新的内容判断
- 不修改任何非杂质的表达
- 原始结构保留

## 后续更新

- 发现新杂质类型时，补充到杂质定义表
- 根据后续处理反馈，调整杂质定义
- 每处理一份新稿子后，对照检查是否遗漏新模式
